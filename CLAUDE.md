# VoiceCanvas 项目规范

> 本文件仅保留项目分支、提交与 PR 规范。具体产品设计与实现方案以通过 brainstorm 后产出的设计文档为准。

---

## 分支与提交规范

### 分支命名

```
<类型>/<简短描述>
```

类型：
- `feat/`：新功能
- `fix/`：修复
- `docs/`：文档
- `chore/`：构建/配置
- `refactor/`：重构

### Commit 消息格式

```
<type>: <简短描述>

<可选的详细说明>
```

示例：
```
feat: 支持语音绘制矩形

- 添加 RectangleCommand 解析
- 在 Canvas 上绘制指定位置/颜色/尺寸的矩形
- 更新 README 指令列表
```

### PR 描述模板

每个 PR 描述必须包含以下四部分：

```markdown
## 功能描述
本 PR 实现了什么功能，用户如何使用。

## 实现思路
技术选型和核心逻辑说明。

## 测试方式
如何验证该功能正常运行（语音指令示例 + 预期结果）。

## 相关 Issue
无 / 关联需求编号
```

---

## 比赛规则提醒

- 所有 commit 时间戳必须在比赛周期内。
- 每个 PR 只做一件事，保持小粒度。
- 每次 PR 合并后主分支必须可运行。
- 第三方依赖和代码复用需在 README / PR 中声明。
